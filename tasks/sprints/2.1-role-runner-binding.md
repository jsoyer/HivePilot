   binary is not found (shutil.which), the runner should raise a clear, catchable error

   pick the lowest-friction option consistent with how the repo loads other yaml. If you load
   it, add a loader; if not, keep roles.py values authoritative and the yaml documentary ‚Äî
   STATE which you chose in the return summary.

   - ceo has dual models `["qwen", "kimi"]`.

   - existing `tests/test_roles.py` still passes (don't break existing role assertions).
