## Tasks
1. **RunStatus transitions:** In `run_pipeline`, persist the lifecycle using the `RunStatus`
   enum values (string-valued, so pass `RunStatus.RUNNING.value` etc. to existing
   state_service functions): start → `RUNNING`; on stage failure (no continue) →
   `TEST_FAILURE`; all stages ok → `COMPLETE`. Keep using existing `record_run_start` /
   `complete_run` — do NOT change their signatures; just pass RunStatus enum `.value`.
2. **Per-stage artifact:** After each stage executes, call
   `pipelines.write_stage_artifact(vault_path=<resolved vault or None>, run_id=run_id,
   stage_name=stage.name, output=<short stage summary str>, dry_run=<dry_run flag>)`.
   Resolve `vault_path` from settings/config the same way the rest of the codebase does
   (grep for how obsidian vault path is obtained — it's read-only, just read it). If no
   vault configured, pass `vault_path=None` (function returns None — that's fine).
   **Default dry_run=True.**
3. **Stage → role (Claude-first):** A `PipelineStage` currently only has `task`. Map a stage
   to a role by stage/task name when a matching role exists (`get_role`); otherwise proceed
   as today. For the runner choice, default to `claude` when no binding exists. Keep this
   minimal — Sprint 2.1 will own the full role→runner+model table. Do NOT implement the
   binding table here; just leave a clean seam (e.g. a helper `_runner_for_stage(stage)`
   that returns `"claude"` for now).
4. **CLI:** Ensure `run-pipeline` (`cli.py:184-212`) passes a `--dry-run/--no-dry-run`
   flag (default dry-run TRUE) through to `run_pipeline`, controlling vault writes only.
   If `run_pipeline` needs a new `dry_run: bool = True` kwarg, add it (additive, defaulted).
5. **Tests** (`tests/test_pipeline_execution.py`): 
   - run_pipeline persists RUNNING then COMPLETE for a happy-path pipeline (mock runners).
   - on a failing stage (no continue_on_failure), final status is TEST_FAILURE and later
     stages don't run.
   - per-stage artifact is attempted once per stage; with `vault_path=None` it's a no-op
     (returns None) and with a tmp vault + dry_run=False the file exists under
     `12 - HivePilot/Runs/`.
   - dry-run default: vault not written when dry_run=True.
