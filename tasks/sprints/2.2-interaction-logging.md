## Design (keep simple, immutable, English)
1. `@dataclass(frozen=True) Interaction`: `actor: str`, `action: str`, `target: str | None`,
   `summary: str`, `timestamp: str` (ISO str passed in ‚Äî do NOT call datetime.now in core
   logic; accept it as a param so tests are deterministic), `run_id: int | None = None`,
   `metadata: dict | None = None`.
2. `class InteractionService`:
   - `__init__(self, vault_path: Path | str | None, dry_run: bool = True)` ‚Äî wraps an
     `ObsidianService` when vault_path is truthy/exists, else writes are no-ops returning None.
   - `log_interaction(self, interaction: Interaction) -> dict | None` ‚Äî writes ONE note to
     `Interactions/{timestamp-date}-{actor}-{slug(action)}.md` with frontmatter
     (actor, action, target, run_id, timestamp) via `write_note`. Returns the emit dict
     (or None if no vault).
   - `render_timeline(self, interactions: list[Interaction]) -> str` ‚Äî returns a **Mermaid**
     fenced block. Use a `sequenceDiagram` (actor->>target: action) OR mermaid `timeline`
     syntax; pick one and be consistent. Pure function, no I/O.
   - `write_timeline_note(self, interactions: list[Interaction], timestamp: str) -> dict | None`
     ‚Äî writes a note `Interactions/{date}-timeline.md` whose body embeds `render_timeline(...)`.
3. dry_run default TRUE everywhere; no real vault write unless dry_run=False AND vault exists.

## Tests (`tests/test_interaction_service.py`) ‚Äî use a tmp vault dir
- `render_timeline` returns a string containing a ```` ```mermaid ```` fence and each

- `write_timeline_note` body contains the mermaid block.
