## Objective

Replace `notification_service.send_notification()`'s static `if/elif slack/discord/telegram` dispatch with a `NotifierRegistry`/`NOTIFIER_MAP` (mirroring Sprint 1's `RunnerRegistry`/`RUNNER_MAP` pattern), make the 3 built-in channels self-register at import, and wire `PluginManager.declared_notifiers` (collected in Sprint 2 but not yet acted on) into the new registry so a plugin notifier is actually invoked for pipeline events.

## Why this exists (context, no PRD lookup needed)

`hivepilot/services/notification_service.py` currently has (lines ~51-65):

```python
def send_notification(message: str, channels: Iterable[str] | None = None) -> None:
    channels = list(channels) if channels else ["slack", "discord", "telegram"]
    for channel in channels:
        channel = channel.lower()
        try:
            if channel == "slack":
                _send_slack(message)
            elif channel == "discord":
                _send_discord(message)
            elif channel == "telegram":
                _send_telegram(message)
        except _NotConfigured:
            pass  # silently skip unconfigured channels
        except Exception as exc:  # noqa: BLE001
            logger.warning("notification.failed", channel=channel, error=str(exc))
```

`_send_slack`, `_send_discord`, `_send_telegram` are already separate functions with signature `Callable[[str], None]` (well, `_send_telegram` additionally accepts optional kwargs for a richer streaming path used by `stream_agent_turn` — **that richer path is a separate code path and out of scope**, see below). `_NotConfigured` is a private exception each `_send_*` raises when its webhook/token env var is missing.

**Important, verified fact:** `send_notification()` is **not currently exercised by any existing test** — `tests/test_notification_service.py`'s tests all target `stream_agent_turn`, which calls `_send_telegram` **directly**, bypassing `send_notification()` entirely (a separate, richer "stream a live agent turn as a Telegram card" feature). This means refactoring `send_notification()`'s dispatch cannot break those existing tests — verify this remains true by grep before you start: `grep -n "send_notification(" tests/test_notification_service.py` should return nothing.

## Tasks

- [ ] In `hivepilot/services/notification_service.py`, add near the top (after existing imports, before `send_notification`): `NOTIFIER_MAP: dict[str, Callable[[str], None]] = {}` and `class NotifierKindCollisionError(RuntimeError): pass`.
- [ ] Add `class NotifierRegistry:` with `@staticmethod def register(name: str, fn: Callable[[str], None], *, override: bool = False) -> None:` — same collision logic as `RunnerRegistry.register` (Sprint 1): if `name in NOTIFIER_MAP and NOTIFIER_MAP[name] is not fn and not override: raise NotifierKindCollisionError(f"Notifier '{name}' is already registered to {NOTIFIER_MAP[name].__name__}; refusing to silently replace it")`; else `NOTIFIER_MAP[name] = fn`. Add `@staticmethod def known_names() -> frozenset[str]: return frozenset(NOTIFIER_MAP)`.
- [ ] Rename the existing `_NotConfigured` exception's public-facing alias: add `NotConfigured = _NotConfigured` right after the `_NotConfigured` class definition (keep `_NotConfigured` itself unchanged and unremoved — it's still used internally by `_send_slack`/`_send_discord`/`_send_telegram`; `NotConfigured` is simply the same class, publicly importable, so a plugin notifier can `from hivepilot.services.notification_service import NotConfigured` and raise it).
- [ ] After the three `_send_slack`/`_send_discord`/`_send_telegram` function definitions (do not change their bodies or signatures), add self-registration: `NotifierRegistry.register("slack", _send_slack)`, `NotifierRegistry.register("discord", _send_discord)`, `NotifierRegistry.register("telegram", _send_telegram)` — note `_send_telegram`'s full signature has extra optional kwargs (`chat_id`, `message_thread_id`, `parse_mode`) beyond `Callable[[str], None]`; that's fine, Python doesn't enforce the narrower `Callable` type at runtime and `send_notification()` only ever calls it with the single positional `message` argument, exactly as today.
- [ ] Rewrite `send_notification()`'s body: keep the `channels = list(channels) if channels else ["slack", "discord", "telegram"]` default and the `channel.lower()` normalization, but replace the `if/elif` with: look up `fn = NOTIFIER_MAP.get(channel)`; if `fn is None`, `logger.warning("notification.unknown_channel", channel=channel)` and `continue` (a channel name with no registered notifier — built-in or plugin — is reported, not silently ignored, and does not raise); otherwise call `fn(message)` inside the *same* existing `try/except _NotConfigured: pass` / `except Exception as exc: logger.warning(...)` structure (also catch `NotConfigured` — since it's now the same class as `_NotConfigured`, `except _NotConfigured` alone already covers both names).
- [ ] In `hivepilot/plugins.py`, import `NotifierRegistry` and `NotifierKindCollisionError` from `hivepilot.services.notification_service` at the top of `PluginManager.__init__` (or module level — avoid a circular import: check whether `notification_service.py` imports anything from `plugins.py` — it should not; if it doesn't, a top-of-file import in `plugins.py` is fine and preferred for consistency with the `RunnerRegistry` import style from Sprint 2). Replace the Sprint-2 placeholder (`self.declared_notifiers.update(...)`) with: after building `self.declared_notifiers` from all plugins' `"notifiers"` dicts (still needed for the `plugins list` provenance in Sprint 4 — keep populating it), also call `NotifierRegistry.register(name, fn)` for each entry, letting `NotifierKindCollisionError` propagate uncaught (same hard-stop-on-collision policy as runner kinds).
- [ ] Add `tests/test_notifier_registry.py` covering: (a) `NOTIFIER_MAP` contains `slack`/`discord`/`telegram` mapped to `_send_slack`/`_send_discord`/`_send_telegram` immediately after `import hivepilot.services.notification_service`; (b) registering a new notifier name succeeds and `send_notification("hi", channels=["my-channel"])` calls it; (c) registering an existing name with a different callable and no `override` raises `NotifierKindCollisionError`; (d) `send_notification` with an unregistered channel name logs a warning and does not raise; (e) a plugin notifier registered via `PluginManager` (construct a `PluginManager` against a temp `plugins/` dir with a fixture `register()` returning `{"notifiers": {"custom": lambda msg: ...}}`, monkeypatching `settings.base_dir`) is present in `NOTIFIER_MAP["custom"]` after construction, and `send_notification("hi", channels=["custom"])` invokes it.

## Acceptance Criteria

- [ ] `NOTIFIER_MAP` contains `slack`, `discord`, `telegram` immediately at import time, mapped to the original functions (no behavior change for existing callers of `send_notification()`).
- [ ] `send_notification()` dispatches through `NOTIFIER_MAP`, not an `if/elif` chain.
- [ ] A plugin notifier (registered via `PluginManager`, either loading mechanism from Sprint 2) is invoked by `send_notification()` when its channel name is requested.
- [ ] A notifier-name collision (plugin vs. built-in, or plugin vs. plugin) raises `NotifierKindCollisionError` and is not silently absorbed.
- [ ] An unregistered channel name passed to `send_notification()` logs a warning, does not raise, and does not silently do nothing without a trace.
- [ ] `tests/test_notification_service.py` (existing, `stream_agent_turn`-focused) passes unmodified.
