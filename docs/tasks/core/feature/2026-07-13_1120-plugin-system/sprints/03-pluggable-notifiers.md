## Objective

Replace `notification_service.send_notification()`'s static `if/elif slack/discord/telegram` dispatch with a `NotifierRegistry`/`NOTIFIER_MAP` (mirroring Sprint 1's `RunnerRegistry`/`RUNNER_MAP` pattern), make the 3 built-in channels self-register at import, and wire `PluginManager.declared_notifiers` (collected in Sprint 2 but not yet acted on) into the new registry so a plugin notifier is actually invoked for pipeline events.

## Why this exists (context, no PRD lookup needed)

`hivepilot/services/notification_service.py` currently has (lines ~51-65):

```python
def send_notification(message: str, channels: Iterable[str] | None = None) -> None:
    channels = list(channels) if channels else ["slack", "discord", "telegram"]
    for channel in channels:
        channel = channel.lower()
        try:
            if channel == "slack":
                _send_slack(message)
            elif channel == "discord":
                _send_discord(message)
            elif channel == "telegram":
                _send_telegram(message)
        except _NotConfigured:
            pass  # silently skip unconfigured channels
        except Exception as exc:  # noqa: BLE001
            logger.warning("notification.failed", channel=channel, error=str(exc))
```

`_send_slack`, `_send_discord`, `_send_telegram` are already separate functions with signature `Callable[[str], None]` (well, `_send_telegram` additionally accepts optional kwargs for a richer streaming path used by `stream_agent_turn` — **that richer path is a separate code path and out of scope**, see below). `_NotConfigured` is a private exception each `_send_*` raises when its webhook/token env var is missing.

**Important, verified fact:** `send_notification()` is **not currently exercised by any existing test** — `tests/test_notification_service.py`'s tests all target `stream_agent_turn`, which calls `_send_telegram` **directly**, bypassing `send_notification()` entirely (a separate, richer "stream a live agent turn as a Telegram card" feature). This means refactoring `send_notification()`'s dispatch cannot break those existing tests — verify this remains true by grep before you start: `grep -n "send_notification(" tests/test_notification_service.py` should return nothing.
